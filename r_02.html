Data

In the previous section you were asked to perform some mathematical operations in R. This involved using data values and mathematical operators. The data were all numerical values, but R handles more than just numerical data. It actually considers and categorises data values as any of six type, four of which we encounter on a regular basis.

Data type

The category of data values is defined as the type of the data. Data type is important to consider as it has strong effects on how R behaves and understands the data it works with. The four most important types to be aware of are logical, integer, double and character. The type of data can be returned by passing the data to the typeof() function, e.g. typeof(12).

Logical data

The most basic of these four is logical type data. For this type of data, elements can be one of two values: TRUE and FALSE. You will often see that T and F are used as synonyms; this is a bad idea as the meaning of T and F can be over-written, e.g. T <- FALSE makes T mean FALSE, which can wreak havoc. Do not use T and F as synonyms for TRUE and FALSE. It will be especially important to think about logical data when we perform logical operations to subset datasets.

TRUE
    FALSE
    typeof(TRUE)
    typeof(FALSE)
## [1] TRUE
    ## [1] FALSE
    ## [1] "logical"
    ## [1] "logical"
Integer data

The next data type to consider is integer data. As the name suggests, this is a numerical data type of whole numbers (or “integers”). These are the sorts of numbers we use for counts, phone numbers, points, etc. - data where we cannot have fractions. To create integer values in R, we provide the number suffixed by an upper case L; this tells R to treat the number as an integer. Because of the way computers handle data, integers in R can be values between approximately -2.15B and 2.15B; we could not express the world population as an integer (at the time of writing, about 7,866,319,070… apparently.

1L
    2L
    typeof(3L)
    typeof(7866319070L) == "integer"
## [1] 1
    ## [1] 2
    ## [1] "integer"
    ## [1] FALSE
Double data

Double type data is the other numeric type of data in R. This is numbers where values can (but might not) have fractions. This is used for number values such as distances, weights, temperatures etc. These data are given simply by providing the numerical value; R assumes it to be double type even when the number is a whole number.

3.14
    4.2994881
    typeof(12.44)
    typeof(1)
## [1] 3.14
    ## [1] 4.299488
    ## [1] "double"
    ## [1] "double"

An exception to this assumption is when we produce a sequence of numbers using the :. In this case, R will produce a sequence of integer values.

1:3
    typeof(1:3)
## [1] 1 2 3
    ## [1] "integer"
Character data

The final one of the four key type is character type data. This is data where values can contain letters, numbers and special characters. We show R that data is character type by wrapping it in quotations. Either double- or single-quotes - " and ' can be used, but the data must be wrapped in a matching pair (e.g. "Hello' would not work). My preference is double to prevent issues when handling text strings that contain '. A single element of character data is called a “string”.

"Hello"
    'This is a string'
    "I've got a single-quote in this string."
    typeof("Hello")
    typeof("1")
## [1] "Hello"
    ## [1] "This is a string"
    ## [1] "I've got a single-quote in this string."
    ## [1] "character"
    ## [1] "character"
