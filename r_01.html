R

R is a programming language widely used for data-oriented tasks. It was designed specifically with statistical programming in mind and is popular among statisticians, data scientists and researchers. It is well suited to tasks such as sorting, filtering and joining datasets, modifying, deriving and processing variables, producing summary statistics, figures, tables and datasets, and performing complex statistical analyses. Thanks to add-ons such as RMarkdown and Shiny, it can also be used to produce reports in html, pdf and word formats, presentations, tutorials and quizzes, websites and web-based apps all with R code intertwined into their development.

R is used to write source code; code written in a human-readable or “high-level” programming language, providing instructions to the computer. Computers will then read this source code and perform the tasks it specifies. As an example, we may wish to simulate a simple coin-toss experiment. The code should simulate 20 flips of a coin and return the number of times heads occurs. Source code to perform this simulation could be written in R as:

sum(sample(x=c("H", "T"), size = 20, replace = TRUE)=="H")

That code would make the computer perform three processes. First it would generate 20 outcomes by randomly sampling from the values H and T. The computer would compare each of those 20 values to the value H returning a value of true or false for each case. Finally it would return the number of trials where the outcome was heads (because sum() will interpret logical values and 1 or 0 for true or false).

The R interpreter

The R code for a coin toss simulation is a fairly simple piece of code to read. You can probably figure out what some parts of that code are doing even without knowing R. For example, you can probably guess that sample() is telling the computer to do some sampling, size is telling the computer what the sample size should be and replace is telling it to sample with replacement. The reason it is easy to read that code is because R is a “high-level” programming language. This means it is designed to be very human-readable. Unfortunately, while being human-readable is a good thing for programmers, that means computers do not understand it all; computers understand machine code (typically binary). To make source code usable, we have to translate it into machine code.

There are three main approaches to translating source code. Compiled and assembled languages both use software, a compiler or assembler respectively, to create an executable file of machine code. Executable files can then be run on computers without the source code and compiler/assembler present. Interpreted languages on the other hand use software called an interpreter. With interpreted languages, such as R, an interpreter receives source code and translates it on execution. This is useful for interactive programming, where source code is developed or tasks performed on the fly, but requires both source code and an interpreter to be present if R is to be used.

Before you can start using R and learning the language, you first need to set up with the R interpreter. The R interpreter can be downloaded from the CRAN (Comprehensive R Archive Network) homepage and installed on your computer. This will install the interpret and an application called the console. The console is a command-line style terminal used to interact with the interpreter; we type and submit R code here, which the interpreter communicates to the computer, and the results are returned. Install the R interpreter, open the console, type print("hello, world!") in the bottom line (next to the >) and press enter. Here you used print() to make the computer print the value hello, world! to the console.

Mathematical operations

Among the most basic tasks to perform with R are simple calculations. All we need to perform these in R are numbers and mathematical operators, all of which can be found on your keyboard. Mathematical operators are just special characters that tell the computer to perform normal mathematical tasks such as addition, subtraction etc.. The most important are:

  • + adds; x+y is x plus y
  • - subtracts; x-y is x minus y
  • * multiplies; x*y is x times y
  • / divides; x/y is x divided by y
  • ^ exponentiates; x^y is x to the power of y

R will perform mathematical operations in the standard order of operations used in mathematics; operations within parentheses, then exponentiation, then multiplication and division (from left to right), and finally addition and subtraction (also from left to right). Parentheses, ( and ), are always treated in this way in R; it will perform any processes within parentheses first. To practice using the console and learn the mathematical operators, write and submit source code to calculate:

  • Four times six
  • 18 divided by two
  • Five plus six, times 12
  • Five times three, plus four-squared

Each expression should be written then submitted, pressing enter, before writing the next. Results should have been returned to the console on the line immediately below the executed code. You should have written and submitted four lines of code that look something like this:

4*6
    18/2
    (5+6)*12
    (5*3)+4^2

Sometimes you will see that the last line has + at the left side (where the > is normally) and output has not been returned; this means you have submitted an incomplete expression and the interpreter is waiting for you to finish. Submitting the below code would cause this as the closing parenthesis is missing. You can solve this by submitting the missing code, ), or pressing esc. To see, resubmit and edit previously submitted code, navigate up (and down) using the keyboard arrows in the console.

RStudio

While the console included with the R interpreter is a useful tool for quick and simple interactive programming, you can probably imagine that working by typing code directly into the console becomes less viable for more complex projects. To make life easier, programmers will use an editor. An editor is simply a plain-text document where we write, edit and develop code to perform tasks. While any text editor, even Word, could do this, it is best to use something g designed for the purpose. The R application that came bundled with the interpreter, the same application where we find the console includes an editor. Open the R application and go to file > new script. You should now have both the console and an editor open. Type the following expressions into the editor, putting each on to a new line. Run each line by putting the cursor on the line, or highlighting the whole expression, and pressing ctrl + R.

  • Divide 17 minus five by two
  • Multiply 42 plus 16 by five over six (five-sixths)
  • Two to the power of three to the power of four

As with the console, each new expression should be on a new line. Pressing enter now adds a line but does not execute the code, instead the code is kept in the editor and can be repeatedly run, read and edited. Pressing ctrl + R with the cursor on a lone that contains code, or when code is highlighted, causes that code to execute. Your script should look like the following:

(17-5)/2
    (42+16)*(5/6)
    2^3^4
## [1] 6
    ## [1] 48.33333
    ## [1] 2.417852e+24

Note that white space (between data values, operators, names etc.) is not important in R. You could very much write these as, for example, (42 + 16) * (5 / 6) if you want, but (4 2 + 1 6) * (5 / 6) would not work as there are spaces within values. White space should be used to make code more readable and neatly arranged.

The R editor is more useful than the console for serious development, but even this isn’t the tool that most programmers are using. Most are using third party applications as development environments. Perhaps the most popular of these among R programmers is RStudio. (RStuido can be downloaded)[#] and installed to run locally. You can also run RStudio through a browser on the RStudio Cloud. Both of these are great options and which you prefer is up to you - local installation takes away the reliance on internet connects allowing you to work on the move, while RStudio Cloud means your programs can be accessed (because they are saved on the RStudio Cloud) and run by logging into any computer. Cloud storage of programs can also be managed by using Dropbox, Drive, iCloud etc. meaning you can get the best of both. Sometimes you may not have the rights to install applications on your computer so RStudio Cloud may be your only option. If you can, download and install RStudio now or create an account and login to RStudio Cloud, then open RStudio.
