Intro to R

R is a programming language widely used for data-oriented tasks. It was designed specifically with statistical programming in mind and is popular among statisticians, data scientists and researchers. It is well suited to tasks such as sorting, filtering and joining datasets, modifying, deriving and processing variables, producing summary statistics, figures, tables and datasets, and performing complex statistical analyses. Thanks to add-ons such as RMarkdown and Shiny, it can also be used to produce reports in html, pdf and word formats, presentations, tutorials and quizzes, websites and web-based apps all with R code intertwined into their development.

R is used to write source code; code written in a human-readable or “high-level” programming language, providig instructions to the computer. Computers will then read this source code and perform the tasks it specifies. As an example, we may wish to simulate a simple coin-toss experiment. The code should simulate 20 flips of a coin and return the number of times heads occurs. Source code to perform this simulation could be written in R as:

sum(sample(x=c("H", "T"), size = 20, replace = TRUE)=="H")

That code would make the computer perform three processes. First it would generate 20 outcomes by randomly sampling from the values H and T. The computer would compare each of those 20 values to the value H returning a value of true or false for each case. Finally it would return the number of trials where the outcome was heads (because sum() will interpret logical values and 1 or 0 for true or false).

The R interpreter

The R code for a coin toss simulation is a fairly simple piece of code to read. You can probably figure out what some parts of that code are doing even without knowing R. For example, you can probably guess that sample() is telling the computer to do some sampling, size is telling the computer what the sample size should be and replace is telling it to sample with replacement. The reason it is easy to read that code is because R is a “high-level” programming language. This means it is designed to be very human-readable. Unfortunately, while being human-readable is a good thing for programmers, that means computers do not understand it all; computers understand machine code (typically binary). To make source code useable, we have to translate it into machine code.

There are three main approaches to translating source code. Compiled and assembled langauages both use software, a compiler or *assembler respectively, to create an executable file of machine code. Executable files can then be run on computers without the source code and compiler/assembler present. Interpreted languages on the other hand use software called an interpreter. With interpreted languages, such as R**, an interpreter receives source code and translates it on execution. This is useful for interactive programming, where source code is developed or tasks performed on the fly, but requires both source code and an interpreter to be present if R is to be used.

Before you can start using R and learning the language, you first need to set up with the R interpreter. The R interpreter can be downloaded from the CRAN (Comprehensive R Archive Network) homepage and installed on your computer. This will install the interpret and an application called the console. The console is a command-line style terminal used to interact with the interpreter; we type and submit R code here, which the interpreter communicates to the computer, and the results are returned. Install the R interpreter, open the console, type print("hello, world!") in the bottom line (next to the >) and press enter. Here you used print() to make the computer print the value hello, world! to the console.
