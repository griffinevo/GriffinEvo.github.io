Data Type in R

One of the most important fundamental aspects to understand with R is data type. This matters because it determines the behaviour of R, and therefore the behaviour of the program and the programmer. A thorough understanding of data types can reduce the number of errors that arise, make it easier diagnose the cause of errors and improve the elegance and efficiency of code. In R there are four types of data that most users encounter; others (raw and complex) do exist but are much more rarely encountered so aren't covered here. The four data types to pay attention to are:

  • Logical
  • Integer
  • Double
  • Character

Note that in this section I use the c() function to create vectors of data values and the assignment operator -> to store those vectors in the workspace, making it easier to reuse data. I also use functions to explore data type, such as typeof() (which returns the data type of the provided vector). Understanding these is not essential for these notes but more can be read about them in other sections.

Logical

Logical-type data is the mot basic of the four covered here. It can be one of two values; TRUE or FALSE. Vecotrs of logical values are very useful in indexing data structures as shown in the data class section The following creates a vector of four elements of logical-type, as shown by the typeof() function. Note that it is the vector that has a type, which is determined by the values it contains, which is why one value - "logical" is returned rather than four - "logical" "logical" "logical" "logical".

v_log <- c(TRUE, FALSE, FALSE, TRUE)
typeof(v_log)

## [1] "logical"

Logical-type data has two values, TRUE and FALSE, but these can be substituted for T and F respectively. This is commonly seen when providing options on functions, e.g. mean(..., na.rm = T). However, this should be avoided and actively discouraged because the values of T and F can be overwritten which could wreak havoc:

# How T and F is often used "in the wild"
mean(c(11, 62, NA, 44), na.rm = T)

# Change the meaning of T
T <- FALSE

# The consequences...
T
TRUE == T
mean(c(11, 62, NA, 44), na.rm = T)

## [1] 39
## [1] FALSE
## [1] FALSE
## [1] NA

It can be very useful to remember that R will convert logical values of TRUE and FALSE to integer values of 1 and 0 respectively when it makes sense to do so. This can make script much more elegant. The below example shows two methods for finding the frequency of the value TRUE, the first is how most would write it, but the other takes advantage of the fact that R will coerce the data to integer values when a function expecting numeric values is used.

# Method 1 uses the length function and logical (==) and mathematical (/) operators
length(v_log[v_log == TRUE]) / length(v_log)
# Method 2 utilises the mean function
mean(v_log)

## [1] 0.5
## [1] 0.5
