Types
The behaviour of R when it handles data depends on the type of data being processed. Data values can be one of six data type, but there are four types that users will meet on a regular basis - logical, integer, double and character. The type of data can be returned using the typeof function.
The simplest of these four main types is logical-type data where values can be one of two values - true or false. The next type is integer-type data, where there is an infinite number of possible values but all those values are integers or "whole numbers" - e.g. counts. The third type is double-type, or numeric-type, which is "real numbers" - numbers that can (but don't necessarily) have decimal places - e.g. lengths in centimetres. The final type, the most complex of the four, is character-type data - data that can (though, again, does not necessarily) contain non-numeric characters - e.g. names. 
typeof(TRUE)
typeof(1L)
typeof(1.0)
typeof('one')

## [1] "logical"
## [1] "integer"
## [1] "double"
## [1] "character"
In the above examples, the logical-type data was the word 'true' written in block capitals - for this data type we can use TRUE and TRUE or T and F, though the latter two are not best practice because T and F can be used as object names so may evaluate to values other than true and false. Integer-type data was specified by giving the integer followed by a capital L. Excluding the L caused R to evaluate the data as double-type. The character-type data was wrapped in quotes - both ' and " can be used. This tells R that the data is character-type. If it were passed as typeof(one) then R would look for an object in the workspace called one, and return the type of data assigned to that object. Character-type data need not be letters either: 
typeof('1')

## [1] "character"
Size
The type of data used should always be the most simple that allows the data to retain all of its information and necessary functionality. For example, if the data was three measurements of 3.2, 4.6, and 5.7, it would make no sense to handle this as logical- or numerical-type - because there is loss of information - and handling this as character-type would make it hard to perform processes like calculating averages
Data should also be kept in the simplest form possible while satisfying the above conditions. This reduces the size of data in the workspace, improving computational efficiency. This is illustrated using the object.size function and the as.* functions. 
object.size(as.logical(TRUE, FALSE, TRUE))
object.size(as.integer(1L:3L))
object.size(as.numeric(1:3))
object.size(as.character('1', '2', '3'))
