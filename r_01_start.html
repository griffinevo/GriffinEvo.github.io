The R programming language

R is a programming language widely used for data-oriented tasks. It was designed specifically with statistical programming in mind and is popular among statisticians, data scientists and researchers. It is well suited to tasks such as sorting, filtering and joining datasets, modifying, deriving and processing variables, producing summary statistics, figures, tables and datasets, and performing complex statistical analyses. Thanks to add-ons such as RMarkdown and Shiny, it can also be used to produce reports in html, pdf and word formats, presentations, tutorials and quizzes, websites and web-based apps all with R code intertwined into their development.

With the R programming language, we can communicate with the computer and have it perform those tasks for us. We provide those instructions in the form of source code; a series of one or more instructions written in the R language. As an example, you may wish to use your computer to simulate a coin toss experiment, where a coin is tossed 10 times and the number of times “heads” and “tails” occur is counted. The source code to perform this simulation and count the number of each possible outcome, using the R language, could be:

table(sample(c("H", "T"), size = 10, replace = TRUE))

You may look at that code and think it’s just meaningless junk to you. That is fine, you are here to learn the R language, not because you know R already. You will soon have the skills needed to read, understand and write much more complex code. Put simply, it told the machine to sample two values - "H" and "T" - ten times with replacement, then return counts of each of the unique outcomes that occurred.

The R interpreter

Computers understand machine-code which, generally speaking, is a stream of binary information in bits, where bits can have a state of on (1) or off (0). The machine reads instructions provided in machine code and performs the processes as instructed. Machine code is a nightmare to program with; it’s hard to read, write and edit, and using it requires a deep understanding of how the computer handles code and instructions. For these reasons, most programmers are using “high-level” languages - those far from machine code - such as Java, Python, and R which handle integration with computer architecture much more automatically. High-level languages are also much easier to use because they are designed to be more human-readable, often even adopting words from human languages. For example, the words length, sum and median in R have meanings which are intuitive to the programmer (assuming the programmer speaks english, of course).

While R and other high-level languages are easier for programmers to use, computers do not understand them. For source code to be executable it must first be translated into machine-readable code. This is generally done in one of two ways. The first uses a compiler program, as used for Java and C++, to convert source code into an executable file of machine code. That executable file can then be run on other machines without the source code and without installing a compiler on that machine; the translation occurs in isolation of execution. The other approach to translation is to use an interpreter, as is the case with R. The interpreter translates code piece-by-piece into machine code as it encountered during execution. Programming with interpreted languages requires that both the source code and an interpreter are present on the machine.

To use the R language on a machine, it must have an R interpreter installed. The interpreter can be downloaded from CRAN (the Comprehensive R Archive Network) and installed. This will download the interpreter and a command-line terminal that allows you to interact with the interpreter called the console. If you do not have the interpreter and console on your computer now is a good time to install it. If you cannot get this installed locally, you can use services like rdrr.io/snippets which gives a console-like input in your browser and runs the code on a server, returning output to the browser; this will do for now as we will quickly move away from using the standard R console.

The console is a command-line terminal where we can type R code and submit it to be executed. Code is typed in the last line of the window, in the line that starts with a >, and is submitted by pressing enter. The computer will perform the requested tasks and return output and messages to the console. Once the code is run, the > will be shown again indicating that the interpreter is ready to receive more input. Copy the code for the coin toss simulator from above and run it in the console; it should return a table showing counts of each outcome.

Mathematical operators

Mathematical operations - the sort of thing you would normally do on a calculator - are among the most basic of processes you can do with R code. You will now use R to write source code that makes your computer perform some mathematical operations for you. Doing this will help you learn how to interact with the R console, see how the execution of R code is ordered and learn some of the mathematical operators available in R. Mathematical operators are just symbols that you will find on your keyboard that do standard things like addition and subtraction:

  • + adds; x+y is x plus y
  • - subtracts; x-y is x minus y
  • * multiplies; x*y is x times y
  • / divides; x/y is x divided by y
  • ^ exponentiates; x^y is x to the power of y

R will perform mathematical operations in the standard order of operations used in mathematics; operations within parentheses, then exponentiation, then multiplication and division (from left to right), and finally addition and subtraction (also from left to right). Parentheses, ( and ), are always treated in this way in R; it will perform any processes within parentheses first. Use numbers, mathematical operators and parentheses to write source code calculating the following:

  • Five plus three
  • Seven minus four
  • Eight times nine
  • Six divided by two
  • Four squared
  • Three times five, plus two
  • Two plus five, times three

Those should expressions should return values of 8, 3, 72, 3, 16, 17 and 21. If that’s not what you got, then check your code resembles the following. Note that you can navigate to previously run code by pressing the up and down keyboard arrows when your cursor is in the console.

5+3
    7-4
    8*9
    6/2
    4^2
    3*5+2
    (2+5)*3

Source files and editors

The R console is a sometimes a very useful tool, but realistically it is not used for more than simple interactive programming. Imagine trying to work by typing your source code directly into the console when you need to perform a complex task such as combining multiple datasets, deriving some variables in the new dataset, and then producing figures and summary statistics. Even if you manage to do that just typing source code directly into the console, all that code is then lost when you end the session (close the console) so, if you needed to perform that task again or modify the outputs, you would have to start from scratch.

Normally programming is done by writing source code into an editor. An editor is simply a plain text document; you could use any text editing software for this such as Notepad, Atom or even Word. However, text editors specifically designed for R code do exist. In the R application that came bundled with the interpreter - the one you opened to access the console- go to file > TODO or press ctrl + n. You should now have both a console and an editor open. Source code can then be written, edited and saved in that file as a .R file. The major benefit here over any other random text editor is that code can be submitted to the console, or run, by pressing ctrl + r; this will run the line the cursor is on or the selected/highlighted code (submiting part of a line, one libne or multiple lines). Write source code to solve the following calculations, putting each statement on a new line. You may need to google for some equations here:

  • 55 times six to the power of two
  • 16 plus five, times 11
  • Convert 15 miles into kilometers
  • Convert 97 degrees Farenheit to Celsius

You should get answers of 21, 1980, 231, approximately 25.1 and approximately 36.1. The source code should look a little something like this:

55*6^2
    (16+5)*11
    1.6093*15
    (97-32)*(5/9)

Note that white-space is generally unimportant in R code, use it to make code more readable. (97-32)*(5/9) is the same as (97 - 32) * (5 /9) and (97-32) * (5/9) but the latter is very reader friendly.
